# GEMINI.md - Instrucciones para el Agente de IA

## Reglas Críticas
1.  **La carpeta `Input/` es INTOCABLE.** Jamás debo modificar, borrar o crear archivos dentro de esta carpeta o sus subdirectorios. Es la fuente de verdad y debe permanecer inalterada. Todo el trabajo se realiza en carpetas de salida, como `david_medina_website/`.

## 1. Mi Rol en este Proyecto
Mi objetivo es asistir en el desarrollo del sitio web personal de David Medina, siguiendo las directrices y la estructura de este proyecto.

## 2. Estructura de la Documentación
Toda la información del proyecto está centralizada en la carpeta `docs/`. **No almacenes información del proyecto aquí.** Este archivo es solo mi punto de partida.

Antes de realizar cualquier tarea, debo consultar los siguientes archivos según sea necesario:

*   `docs/PROJECT.md`: Para entender los **objetivos y el alcance** del proyecto.
*   `docs/PLAN.md`: Para ver el **plan de trabajo** y las fases.
*   `docs/STATUS.md`: Para conocer el **estado actual**, el último avance y el próximo paso a seguir.
*   `docs/DECISIONS.md`: Para consultar el **registro de decisiones** importantes que se han tomado.
*   `docs/DEVELOPMENT.md`: Para obtener la **guía técnica**, incluyendo el stack, la plantilla y las convenciones de desarrollo.

## 3. Flujo de Trabajo
1.  **Consultar `STATUS.md`:** Siempre empezaré por revisar este archivo para saber cuál es la siguiente tarea.
2.  **Ejecutar la Tarea:** Realizaré el trabajo solicitado, basándome en la información de los otros documentos en `docs/`.
3.  **Actualizar `STATUS.md`:** Al finalizar, actualizaré el estado en `STATUS.md`, indicando lo que se completó y cuál es el nuevo próximo paso.
4.  **Registrar Decisiones:** Si durante el proceso se toma una nueva decisión de arquitectura o diseño importante, la registraré en `docs/DECISIONS.md`.
